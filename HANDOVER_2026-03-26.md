# FreeDV 任務交接文件（2026-03-26）

## 1. 交接摘要

本輪主要工作聚焦在「背景收音 -> 回前景後延遲分析（deferred decode）」流程的穩定化與 UX 重整，並同步移除部分舊功能（Power Profile、Haptic、首頁 debug log、Live Activity 啟動邏輯）。

核心目標：

1. 背景收音資料可在前景分析，且可暫停/續跑/取消。
2. 分析與即時 RX stop 的交互不互相踩壞。
3. 分析過程若 App 退到背景，能自動安全暫停。
4. 背景回前景可累積「多筆分析任務」，用清單管理。
5. 日誌與地圖的座標資料不再誤導（背景回放解碼事件不附座標）。

## 2.4 背景分析資料座標修正

問題：背景回放解碼時，事件附上當前位置會誤導（實際收訊地點不可得）。

修正：

- 在 `AudioManager` 的 `onCallsignDecoded` 內，若為 deferred replay：
  - `latitude/longitude` 強制寫入 `nil`
  - map 自然不會顯示該事件座標

## 2.5 Session 秒數顯示偏大修正（3 秒 grace 被算進去）

問題：log 清單顯示的 duration 常多約 3 秒（unsync grace period）。

修正：

- 新增「實際失去 sync 時間」記錄，session finalize 時用該時間當 `endTime`。
- `ReceptionLogger.endSession()` 改為可傳入 `endTime`。

結果：

- duration 更貼近實際收到訊號的時間，不再固定多算 grace 秒數。

## 2.6 STOP 與背景分析競態修正（多次調整）

已處理的情境：

1. 回前景自動分析中，按 STOP 不應讓分析狀態錯亂。
2. 使用者需求更新：按 STOP 應停止「即時解碼」，背景分析可持續。

目前行為：

- 若 deferred analysis 正在跑：
  - STOP 只停即時 RX 路徑（audio engine / tap）。
  - 不主動 cancel deferred analysis。
  - 避免 teardown 掉分析仍在使用的資源。
- 若 deferred analysis 沒在跑：
  - STOP 走完整 teardown + reset。

## 2.7 Lifecycle：分析途中退背景自動暫停

新增機制：

- 監聽 `UIApplication.willResignActiveNotification`。
- 前景分析進行中若 app 失焦（切 app/回桌面），自動 pause。
- 回前景後維持 paused，需使用者手動 resume。

## 2.8 背景分析 GUI 改為任務清單（Queue）

需求背景：

- 使用者可在未 stop RX 的情況下多次進出背景，形成多次背景收音，回前景後應有多筆分析任務。

目前實作：

- ViewModel 新增 `backgroundAnalysisTasks`（含 `pending/running/paused/completed/cancelled` 狀態）。
- 每次從背景回前景且偵測有背景收音（`rxChunkCount > 0`）時，新增一筆 `Capture #n` 任務。
- 分析頁改為列表顯示每筆任務的：
  - 狀態
  - 進度
  - scanned 時間
  - ETA
- 控制區：
  - `Start Next Analysis`
  - `Pause/Resume Current Analysis`
  - `Cancel Current Analysis`

## 4. 關鍵資料流與狀態機（交接重點）

## 4.1 Deferred 分析控制（AudioManager）

主要旗標：

- `deferredDecodeInProgress`
- `deferredDecodePaused`
- `deferredDecodeActive`（lock 保護）
- `deferredDecodePauseRequested`
- `deferredDecodeCancelRequested`
- `deferredReplayFastMode`

關鍵 API：

- `decodeDeferredSamples()`：啟動/恢復 deferred 分析
- `setDeferredDecodePaused(_:)`：暫停/續跑
- `cancelDeferredDecode()`：取消分析
- `stop()`：依是否有 deferred active 走不同 teardown 分支

## 4.2 ViewModel 任務清單（UI 層）

模型：

- `BackgroundAnalysisTaskStatus`
- `BackgroundAnalysisTask`
- `backgroundAnalysisTasks: [BackgroundAnalysisTask]`

關鍵方法：

- `enqueueBackgroundAnalysisTask()`
- `ensurePendingTaskExists()`
- `syncBackgroundAnalysisTasks()`
- `startDeferredDecodeAnalysis()`
- `cancelDeferredDecodeAnalysis()`

## 4.3 Background/Foreground 交互

`enterBackground()`：

- 關閉前景 UI 負載，啟用背景收音。
- 若分析進行中 -> 自動 pause。

`exitBackground()`：

- 根據 heartbeat 判斷是否有背景收音，若有則新增 queue 任務。
- 若剛才是分析中被切走 -> 保持 paused。
- 否則沿用「回前景觸發分析」策略。

## 6. 已知限制 / 潛在風險

## 6.1 任務清單目前偏 UI queue，實際底層仍是單一 deferred sample 檔案

- 現在任務清單可累積「任務項目」，但底層 `DeferredSampleStore` 目前仍是單一 `deferred_samples.pcm`。
- 若要做到真正「每筆任務對應獨立音訊檔」，需後續把 sample store 升級為多檔/分段儲存模型（例如按背景 session 切檔）。

## 6.2 Live Activity 檔案仍存在

- App 路徑已不再使用，但 extension 檔案仍在。
- 若產品決策為完全移除，後續可清理 target / entitlements / plist 設定。

## 6.3 STOP 行為已符合目前需求，但需再做整合實機驗證

建議重點測試：

1. 回前景自動分析中按 STOP，確認分析持續、RX 停止。
2. 分析中切到其他 app，再回來應為 paused。
3. 多次背景往返後，analysis queue 顯示多筆任務。
4. map 不應顯示 deferred replay 事件座標。
5. log duration 不再多算 3 秒。
